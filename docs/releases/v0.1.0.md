# v0.1.0 Release Evidence

Date: 2026-07-24

Release boundary: one clean-root MVP commit, annotated `v0.1.0`, and the
tag-triggered GitHub Release workflow.

Compatibility boundary: `v0.1.0` supports clean installations only and does not
define compatibility with `v0.1.0-rc.*`.

## Public extraction

The candidate starts from public baseline `b7724ad` and applies a Home-led,
allowlisted framework snapshot. Conflicts use the Home semantics except for the
public `standalone-framework` topology, privacy exclusions, and stable release
metadata.

The sanitized
[extraction manifest](v0.1.0-extraction.json) inventories source, baseline, and
result SHA-256 values plus the `exact-copy`, `public-adaptation`, `delete`, or
`retain-target` disposition for every reviewed public path. The manifest excludes
itself to avoid a recursive hash. It contains no private repository path.
The [single-root MVP supplement](v0.1.0-mvp.json) hash-binds the reviewed
downstream adaptation, privacy hardening, and release-evidence changes layered
onto that extraction and likewise excludes only itself.

The public tree and its rewritten single-root history contain no `career/`, personal identity or attachment,
private provenance, font binary, `.obsidian/`, `runtime/`, `build/`, or
`.career-os/` path. All committed Career records are isolated synthetic test
fixtures.

The final privacy audit checks every reachable Git blob. Resume tests, resume
TeX fixtures/templates, and CI/release workflows are pinned by reviewed
append-only blob hashes. A second local release gate extracts labeled identity,
location, contact, and profile candidates from the private Career Home and
compares them exactly without writing or printing the values. This gate caused
the pre-release multi-commit history to be discarded after it identified real
identity text in an earlier PDF-extraction test.

- Seven distinct Career authorities with schema-2 lifecycles and typed
  cross-authority references.
- Single-note `market.jd` capture and screening; no `market.screening` kind.
- Read-only reviewer subagents and strict JSON packet validators.
- `skills validate-reviewer`,
  `downstream plan/apply/validate/rollback`, and resume commands selected by
  `--resume NAME`.
- Handwritten `\documentclass{career-os}` roots with adjacent `identity.tex`,
  fixed preview/application profiles, and no per-resume JSON manifest,
  font-profile record, or template selector.
- Root English and Chinese homepages plus sixteen tracked Obsidian framework
  assets, including ten paired Workbench Bases.

idempotence, and writes a pure-fiction seven-authority graph:

- verified Work supports an approved Claim;

  `JD 原文` and `重新评价`;

  not-applied Engagement and Decision;

The test validates record schemas and lifecycles, all typed references, matching
Vault-relative host references and wikilinks, both root homepages, all sixteen
static assets, and the absence of generated Bases in the data root.

## Resume gate

The locked Source Han Serif SC and Noto Sans CJK SC fonts were fetched and
verified in ignored local state. English, Simplified Chinese, and multilingual
fixtures each built and preview-exported successfully.

Ubuntu uses the pinned TeX Live 2026 setup action with only the packages required
by the system resume class. Its TeX tree is cached between trusted runs; APT is
used only for the Poppler inspection tools.

Each exported PDF passed the final audit with one page, zero images, no XMP or
unsafe document metadata, no external links, no configured secrets, and no
fixture email or phone in extracted text. Preview compilation included no
avatar.

## Evidence boundary

Automation did not run a real Codex session, Claude Code session, or live
Obsidian interaction. These results establish deterministic framework behavior,
not interactive user-experience acceptance, evidence maturity, career readiness,
application success, or any external career outcome.

Publication pushes the clean-root `main`, waits for its CI to succeed, and only
then pushes the annotated `v0.1.0` tag. CI does not run on tag pushes; the
tag-triggered workflow exclusively owns GitHub Release creation and identity
verification.
