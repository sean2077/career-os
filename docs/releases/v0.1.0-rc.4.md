## Security boundary

Selecting the preview identity projection no longer weakens an
application-policy manifest. Both preview and application exports require the
manifest's exact approved Claim set and exactly one `\CareerClaim` wrapper per
resume bullet before compilation can begin. Preview continues to exclude email,
phone, and the application-only avatar; it is not an evidence-policy escape
hatch.

The Career Home acceptance fixture additionally exercised two real, private
application-policy resume roots. Internal XeLaTeX builds succeeded, while both
real preview and application export attempts failed before producing a PDF
because the migrated material remains draft and its bullets are not wrapped by
approved Claims. No private data or real resume artifact is included in this
public release.

## Publication contract

[`sean2077/career-os`](https://github.com/sean2077/career-os). Publication is
not complete when `main` or the tag is merely pushed. The Release workflow must
check out the tag commit, validate the exact changelog section, re-run the core,
Skill, harness, resume build, and privacy-export gates, and create a non-draft
GitHub prerelease that identifies the same tag. The hosted workflow run and
Release URL are verified after the immutable tag is pushed.
