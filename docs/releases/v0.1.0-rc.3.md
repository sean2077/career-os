## Compatibility and migration impact

The new import interface never scans an uncommitted source or assigns record
state from content. Owners first commit an exhaustive inventory with exactly one
explicit disposition per tracked asset, then review hash-bound transform outputs
before applying a reversible plan. Existing user data is not changed by the
system release.

Resume manifest schema 1 remains accepted. Schema 2 is required only for the
closed multi-file bundle, shared identity, recipient and purpose, watermark,
export ID, and application-only avatar features.

## Clean-room import acceptance

The public test suite covers exhaustive inventory, canonical Git symlink
payloads, unsupported gitlinks, exact and transformed assets, multiple outputs,
Unicode paths, stale commits and plans, destination conflicts, idempotence,
partial failure, provenance, and rollback. No private source path, record,
identity, resume, font, or migration control file is included in this release.

## Resume acceptance

English and Simplified Chinese fixtures compile inside an isolated source
bundle. Preview export excludes contact details and avatars. Synthetic
application tests exercise allowed and rejected roots, dependencies, images,
identity sources, claims, targets, recipients, purposes, links, attachments,
metadata, secrets, existing destinations, path traversal, and symlink escape.

## Publication contract

The public repository is
[`sean2077/career-os`](https://github.com/sean2077/career-os). Publication is
not complete when `main` or the tag is merely pushed. The Release workflow must
check out the tag commit, validate the exact changelog section, re-run the core,
Skill, harness, resume build, and privacy-export gates, and create a non-draft
GitHub prerelease that identifies the same tag. The hosted workflow run and
Release URL are verified after the immutable tag is pushed.
